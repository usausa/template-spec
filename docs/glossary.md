# ドメイン用語集 (glossary)

> このプロジェクトのドメイン用語 (ユビキタス言語) を集約する。**プロジェクトで更新して育てる**辞書。
> AI も人も、REQ / DES / コード / UI で**ここの用語・英語名に一貫して合わせる**。

## 使い方
- 新しいドメイン概念が出たら 1 行足す。**日本語 ・ 英語名 (コード上の名前) ・ 意味**を書く。
- コードの識別子 (クラス / プロパティ / enum 値) は、ここの英語名に一致させる → 命名のブレを防ぐ。
- enum の各値も、意味が非自明なら英語名付きで載せると可読性が上がる (値の一覧そのものはコードが正)。
- 実装が変わったら、語彙・意味が変わった分だけ更新する (現在の型やカラム定義は追わない)。

## 書かないこと (二重管理の回避)
- **コードや DB を見れば分かる情報 (現在の型・カラム・enum の実装値・API の形状) は書かない**。
- ここは「語彙と意味」だけを持つ。現状仕様は生成物 (OpenAPI / スキーマ) とテストが正。
